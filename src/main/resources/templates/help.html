Overview

PLM Migration Assistant documents source-to-Windchill mappings, transformation requirements, lookup/crosswalk needs, validation notes, customer decisions, assumptions, exclusions, and exports. It does not infer mappings, recommend mappings, use AI, or generate executable transformation scripts.

Project dashboard

The dashboard summarizes a migration project, target object tabs, source imports, audit findings, open decisions, and generated documentation. Use the dashboard to open the target-first mapping workspace, run deterministic audit checks, and create exports.

Source schema browser

The source schema browser lists imported source tables/files and their discovered columns. It shows deterministic profile facts, including sample values, null percentage, distinct count, maximum observed length, duplicate indicators, and likely list/enumeration indicators. These facts help an architect make a manual mapping decision; they are not automatic mapping suggestions.

Mapping workspace

The workspace is organized by configurable Windchill target object types. Each tab is loaded from a TargetObjectType record. Select a target tab, review the required Windchill fields, and manually select a source field, mapping type, status, owner, default value, transformation requirement, and validation notes.

Mapping matrix and status

The center matrix is the system of record for target fields in the selected Windchill object type. After you save a mapping, the row remains populated with its selected source field, mapping type, mapping status, owner, default value, and transformation notes.

  • RED: required target field has no source, default value, or not-applicable status.
  • GREEN: mapping status is approved.
  • YELLOW: mapping needs customer review.
  • PURPLE: mapping type is lookup.
  • BLUE: mapping type is default value.
  • GRAY: mapping is not applicable or not migrated.

Detail and audit panel

The right panel explains status colors and summarizes existing mappings for the selected target object tab. It is a quick audit and review area; the editable fields in the center matrix are the primary place to update mapping data in the MVP.

Exports

Exports are generated from structured data already entered in the application. They document mapping specifications, customer review workbooks, validation plans, JSON mapping definitions, and implementation requirements. They do not generate production SQL, executable transformations, or AI-generated pseudocode.

Demo how-to: Agile ITEM to Windchill WTPart

  1. Open the project list and click Create Agile-to-Windchill Demo Project.
  2. Open the generated project dashboard.
  3. Click Open Target-First Mapping Workspace.
  4. Select the WTPart target object tab.
  5. In the source schema browser, review the ITEM fields such as ITEM_NUMBER, DESCRIPTION, and LIFECYCLE_PHASE.
  6. In the number row, select ITEM.ITEM_NUMBER, choose DIRECT, choose APPROVED, add validation notes, and save.
  7. In the name row, select ITEM.DESCRIPTION, choose DIRECT, choose PROPOSED, add a transformation comment if needed, and save.
  8. In the lifecycleState row, select ITEM.LIFECYCLE_PHASE, choose LOOKUP, choose NEEDS_CUSTOMER_REVIEW, and note that the lifecycle crosswalk requires customer approval.
  9. Confirm that saved source fields, mapping types, statuses, owners, defaults, and notes remain visible in the center matrix.
  10. Run the deterministic audit from the project dashboard.
  11. Export the Mapping Specification, Customer Review Workbook, and Implementation Requirements Export.

Troubleshooting mapping status display

If a row stays red after saving, check whether the required row has at least one of these deterministic resolution values: a selected source field, a default value, or a NOT_APPLICABLE mapping status. Also check that the saved mapping appears in the row's Current Mapping column. The input controls should now remain populated after save because the workspace reloads saved mapping data by target attribute.
